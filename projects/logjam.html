Logjam.Next

Logjam.Next was developed specifically for NetApp to address the unique challenges they face in managing and analyzing log data. NetApp, as a leading provider of data management solutions, handles vast amounts of log data from various sources. Logjam.Next aims to enhance NetApp's ability to process and analyze this data efficiently, providing them with valuable insights to improve their products and services. By leveraging the latest technologies and best practices in log processing, Logjam.Next is tailored to meet NetApp's specific requirements, ensuring seamless integration and optimal performance within their existing infrastructure. Key Features

Parallel Processing

Processes nodes in parallel, allowing for faster and more efficient log analysis, especially when handling large volumes of data.

Task Resumption

Picks up the last interrupted task and resumes work from that point. This ensures that log processing tasks can be resumed without loss of progress, even in the event of interruptions or failures.

Abnormal Log Detection

Detects abnormal logs, which is crucial for identifying potential issues or anomalies within the system.

Interactive Web Interface

Provides an interactive web interface, which makes it easier for users to analyze and understand the log data visually.

Flow

Demo

The video above illustrates the flow of Logjam.Next, a log processing tool. Data is initially stored in NFS. If the data is zipped, nodes are extracted. Multiple threads are then utilized to process these nodes in parallel, improving program speed as this is a bottleneck. The extracted nodes are inserted into ElasticSearch using REST APIs.

After all nodes are inserted, the system gains a comprehensive understanding of normal log file patterns. The NetApp Support team can then access the UI, as depicted in the image below, to paste a log and analyze its normality. Users can also specify the platform and versions they are using or leave the default choice as All Platform/Version for general diagnosis.
